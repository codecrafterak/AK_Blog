What You'll Build

This guided lesson shows you how to create a contact form and send submissions directly to your email using EmailJS, without a backend server. Follow the steps, copy the code, and use the live demo structure below to practice.

Project Goal

Prerequisites

üë§

Account

Step 1: Create a free account at EmailJS and log in to your dashboard.

This is your starting point. You'll need access to the EmailJS dashboard to configure services and templates.

Service

Step 2: Add a service (e.g., Gmail or a supported provider) in the Email Services section.

This connects EmailJS to your actual email provider so messages can be delivered.

üìß

Template

Step 3: Create an email template and include variables matching your form fields, like {{name}}, {{email}}, and {{message}}.

Tip: In your template's content, reference variables exactly as your form's input names. For example: "From: {{name}} <{{email}}>" and "Message: {{message}}".

üîë

Public Key

Step 4: Find your public key (sometimes shown as user ID) in your EmailJS dashboard.

This key authenticates your application with EmailJS. It's safe to expose client-side.

EmailJS Setup

üì¶

Step 1: Add EmailJS Script

Include the EmailJS browser SDK before your own scripts. This loads the EmailJS library that provides the functions you'll use.

<script type="text/javascript" 
        src="https://cdn.jsdelivr.net/npm/@emailjs/browser@4/dist/email.min.js">
</script>
üîß

Step 2: Initialize

Call emailjs.init('YOUR_PUBLIC_KEY') once, preferably after the SDK loads. This authenticates your application.

<script type="text/javascript">
  emailjs.init('YOUR_PUBLIC_KEY');
</script>
üÜî

Step 3: Identify IDs

Note your service ID (e.g., default_service) and template ID (e.g., template_qhe68wc).

You'll find these in your EmailJS dashboard. Write them down or copy them somewhere safe - you'll need them in your JavaScript code.

Build the Form

üèóÔ∏è

Create Semantic HTML Form

Create a semantic, accessible form with proper id and name attributes. The name attributes must match your EmailJS template variables.

<form id="contact-form" class="form">
  <div class="form-group">
    <label for="name">Name</label>
    <input type="text" id="name" name="name" required />
  </div>

  <div class="form-group">
    <label for="email">Email</label>
    <input type="email" id="email" name="email" required />
  </div>

  <div class="form-group">
    <label for="message">Message</label>
    <textarea id="message" name="message" rows="5" required></textarea>
  </div>

  <button type="submit" id="send-button" class="btn">
    Send Message
  </button>
</form>

Important: Ensure the form's id and button's id match the ones used in your JavaScript. Use clear, lowercase names.

Connect the Script

üíª

Complete JavaScript Implementation

Listen for form submission, prevent default behavior, show a sending state, and call emailjs.sendForm with your IDs.

<script>
  // Replace with your actual values
  const SERVICE_ID = 'default_service';
  const TEMPLATE_ID = 'template_qhe68wc';
  const PUBLIC_KEY = 'YOUR_PUBLIC_KEY';

  // Initialize EmailJS once
  emailjs.init(PUBLIC_KEY);

  const form = document.getElementById('contact-form');
  const button = document.getElementById('send-button');

  form.addEventListener('submit', function(event) {
    event.preventDefault();

    // Visual feedback
    button.disabled = true;
    const originalText = button.textContent;
    button.textContent = 'Sending...';

    emailjs.sendForm(SERVICE_ID, TEMPLATE_ID, this)
      .then(() => {
        button.textContent = 'Sent!';
        setTimeout(() => {
          button.textContent = originalText;
          button.disabled = false;
        }, 1200);
        form.reset();
        alert('Message sent successfully!');
      })
      .catch((err) => {
        button.textContent = originalText;
        button.disabled = false;
        console.error(err);
        alert('There was an error sending your message. Please try again.');
      });
  });
</script>

Security note: The public key can be exposed client-side, but do not include private API keys. Consider rate limiting or CAPTCHA to reduce spam.

Test and Troubleshoot

‚úÖ

Verify IDs

Double-check your service ID, template ID, and public key. Typos are the most common issue.

Copy these values directly from your EmailJS dashboard to avoid typos. One wrong character will cause the integration to fail.

üîó

Match Fields

Ensure template variables (e.g., {{name}}) exactly match form name attributes.

If your form input has name="email", your template must use {{email}} (not {{user_email}} or anything else).

üì¨

Email Service

Confirm the connected email service works and isn't blocking messages or marking them as spam.

Check your spam folder! Test emails often end up there initially. Also verify your email service is properly connected in EmailJS.

üêõ

Console Logs

Use console.error(err) to see details if an error occurs.

Open your browser's developer tools (F12) and check the Console tab. Error messages here will tell you exactly what went wrong.

üí¨

User Feedback

Provide clear user feedback and disable the button during sending to prevent duplicates.

Users need to know what's happening. Show "Sending..." during submission, "Sent!" on success, and keep the button disabled until the process completes.

üöÄ

Enhancements

Add client-side validation, a success toast, or CAPTCHA (e.g., hCaptcha) to improve reliability and UX.

Once the basics work, consider adding reCAPTCHA to prevent spam, custom success notifications, and more detailed form validation.
